# 🎯 意图识别和深度分析功能

## 新增功能

在思维链可视化的基础上，新增了三个深度分析功能：

### 1. 🎯 目标对齐分析
**检查 LLM 是否真正回答了用户的问题**

- 输出长度是否充足
- 工具是否返回有效数据
- 工具调用成功率

### 2. 🛤️ 最优路径分析
**检查决策链路是否最优**

- 是否有重复调用
- 是否有连续失败
- 是否可以并行化
- 思考效率是否合理

### 3. 🔍 根因分析
**找出导致问题的根本原因**

- 工具失败的根本原因
- 思考轮次过多的根本原因
- 运行时间过长的根本原因
- 重复调用的根本原因

## 快速开始

```bash
# 深度分析
python scripts/analyze_thinking.py .sessions/20260303_132911/trace.jsonl --deep-analysis
```

```
💡 最优路径建议:
   • 添加工具调用重试机制
   • 优化提示词，让 LLM 一次性规划多个工具

🔍 根因分析
====================================================================================================
1. 🔴 工具失败
   现象: get_realtime_data 失败 2 次
   根因: 工具实现问题或参数错误
   解决: 检查工具实现、验证参数格式、添加错误处理

2. 🟡 思考轮次过多
   现象: 15 轮，2 个工具失败
   根因: 工具失败导致 LLM 需要多次重试
   解决: 修复工具失败问题，减少重试次数
```

## 核心价值

**让 LLM Agent 的优化从"凭感觉"变成"有数据支撑"**

- ✅ 快速定位问题：从表面现象到根本原因
- ✅ 理解因果关系：问题之间如何相互影响
- ✅ 获得优化方向：具体的、可执行的建议
- ✅ 持续改进：量化评估优化效果

## 实际案例

**问题**：Agent 运行慢（135.66s），不知道为什么

**深度分析**：
1. 目标对齐：工具未返回有效数据
2. 最优路径：连续失败 + 思考效率低
3. 根因分析：工具实现有 bug → LLM 重试 → 轮次增加

**优化方案**：
1. 修复工具实现（根本原因）
2. 添加重试机制（短期改进）
3. 优化提示词（长期优化）
