# /insight-all 命令使用指南

## 功能说明

`/insight-all` 命令允许你选择任意历史 session 进行 LLM 质量分析，而不仅限于当前 session。

## 命令对比

| 命令 | 分析对象 | 使用场景 |
|------|---------|---------|
| `/insight` | 当前 session | 快速性能检查 |
| `/insight-llm` | 当前 session | 当前 session 质量分析 |
| `/insight-all` | 选择任意 session | 历史 session 质量分析 |

## 完整工作流

### 1. 查看所有 session
```bash
agent >> /sessions
```

### 2. 选择要分析的 session
```bash
agent >> /insight-all
# 使用 ↑↓ 键选择
# Enter 确认
# Esc 取消
```

### 3. 查看分析结果
系统会自动分析选中的 session，提供：
- 决策质量分析
- 效率分析
- 响应质量评估
- 提示词优化建议
- 具体改进措施

## 注意事项

1. **成本**: 会消耗 API token，建议有针对性地分析
2. **Session 选择**: 只显示有 trace.jsonl 文件的 session
3. **取消操作**: 按 Esc 可以取消选择

## 故障排查

### 问题: 选择的 session 没有 trace 数据

```bash
⚠️  Session '20260301_214209' 没有 trace 数据
```

**原因**: 该 session 没有执行任何命令，或 trace 功能未启用

**解决**: 选择其他有数据的 session

### 问题: 没有找到任何 session

```bash
⚠️  没有找到任何 session
```

**原因**: `.sessions/` 目录为空

**解决**: 先执行一些任务创建 session

## 相关命令

- `/sessions` - 查看和切换 session
- `/insight` - 分析当前 session 性能
- `/insight-llm` - 分析当前 session 质量
- `/insight-all` - 分析任意 session 质量
