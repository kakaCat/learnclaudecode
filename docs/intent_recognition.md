# 意图识别和深度分析功能

## 概述

在基础的思维链可视化之上，新增了三个深度分析功能，帮助你全面理解 LLM Agent 的运行质量。

## 三大核心功能

### 1. 🎯 目标对齐分析

**功能**：检查 LLM 是否真正回答了用户的问题

**检查项**：
- ✅ 输出长度是否充足
- ✅ 工具是否返回有效数据
- ✅ 工具调用成功率

**使用场景**：
- Agent 生成了很多内容，但感觉没有回答问题
- 想知道 LLM 是否理解了用户意图
- 评估 Agent 的回答质量

**示例输出**：
```
🎯 目标对齐分析
====================================================================================================

📝 用户问题: 请给我中芯国际的情况

⚠️  发现问题:
   🟡 工具调用未返回有效数据
```

### 2. 🛤️ 最优路径分析

**功能**：检查决策链路是否最优

**检查项**：
- ✅ 是否有重复调用
- ✅ 是否有连续失败
- ✅ 是否可以并行化
- ✅ 思考效率是否合理

**使用场景**：
- Agent 运行太慢，想找出瓶颈
- 想知道如何优化决策链路
- 评估 Agent 的执行效率

**示例输出**：
```
🛤️  最优路径分析
====================================================================================================

⚠️  发现低效模式:
   1. 🔴 连续失败: get_realtime_data (Turn 5 → 6)
   2. 🟡 思考效率低: 15 轮只调用了 14 个工具

💡 最优路径建议:
   • 添加工具调用重试机制
   • 优化提示词，让 LLM 一次性规划多个工具
   • 使用并行工具调用减少轮次
```

### 3. 🔍 根因分析

**功能**：找出导致问题的根本原因

**分析维度**：
- 🔴 工具失败的根本原因
- 🟡 思考轮次过多的根本原因
- 🟡 运行时间过长的根本原因
- 🟡 重复调用的根本原因

**使用场景**：
- 只看到表面问题，不知道根本原因
- 想理解问题之间的因果关系
- 需要针对性的解决方案

## 使用方法

```bash
# 深度分析（包含三个功能）
python scripts/analyze_thinking.py .sessions/xxx/trace.jsonl --deep-analysis

# 深度分析 + 优化建议
python scripts/analyze_thinking.py .sessions/xxx/trace.jsonl --deep-analysis --optimize
```

## 实际案例

**用户问题**：请给我中芯国际的情况

**深度分析结果**：

1. **目标对齐**：❌ 工具调用未返回有效数据
2. **最优路径**：❌ 连续失败 + 思考效率低
3. **根因分析**：工具实现有 bug → LLM 重试 → 轮次增加

**优化方案**：修复工具 → 添加重试机制 → 优化提示词
