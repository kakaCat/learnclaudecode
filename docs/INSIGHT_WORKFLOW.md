# Trace Insight 三个工具使用流程指南

## 📋 工具概览

| 命令 | 分析方式 | 分析对象 | 速度 | 成本 | 适用场景 |
|------|---------|---------|------|------|---------|
| `/insight` | 规则算法 | 当前 session | ⚡ 秒级 | 💰 免费 | 日常快速检查 |
| `/insight-llm` | LLM 分析 | 当前 session | 🐌 分钟级 | 💸 消耗 token | 深度质量分析 |
| `/insight-all` | LLM 分析 | 选择任意 session | 🐌 分钟级 | 💸 消耗 token | 历史对比分析 |

## 🚀 完整使用流程

### 流程 1: 日常开发调试（推荐）

```bash
# 1. 启动 Agent
python -m backend.main

# 2. 执行一些任务
agent >> 帮我分析 backend/ 目录的 Python 文件结构

agent >> 创建一个新的工具文件

# 3. 快速性能检查（免费、秒级）
agent >> /insight

================================================================================

     建议: 考虑并行化、缓存或优化提示词

```

```bash
- 缺少金融专用 API

- 添加提示词模板

```

## 📊 决策树：选择哪个工具？

```
开始
  │
  ├─ 需要快速检查性能？
  │   └─ 是 → 使用 /insight（免费、秒级）
  │
  ├─ 需要深度分析当前任务？
  │   └─ 是 → 使用 /insight-llm（消耗 token）
  │
  └─ 需要对比历史或批量分析？
      └─ 是 → 使用 /insight-all（消耗 token）
```
